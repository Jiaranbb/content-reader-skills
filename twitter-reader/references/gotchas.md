# Twitter/X 踩坑点

## 1. 不要绕开主浏览器登录页

Twitter/X 的正文、回复、浏览数、书签数、X Article 正文常常只在用户主浏览器登录状态下稳定可见。不要为了“隔离”改用未登录页面或公开接口；公开接口缺字段时，以主浏览器可见内容为准。

边界：可以读取页面上可见的内容；不输出、不保存、不复制 cookies、token、localStorage。

## 2. 图文和文章不是同一种排版

普通图文推文：图片放在正文后、内容摘要前的「图片」区域即可。

X Article / 长文章：图片要插入到正文对应段落后，尤其是原文出现「下图」「结果预览」「对照」「架构」「截图」等提示时。不要把所有图片统一堆到末尾。

## 3. 图片必须本地化

远程图片 URL 可能过期或被鉴权拦截。保存图文/文章时，图片必须下载到笔记同级附件目录，并用标准 Markdown 相对路径：

```markdown
![图1](标题附件/01.jpg)
```

不要只保存远程图片 URL；不要默认用 wiki-link 图片语法。

## 4. X Article 的代码和标题要转 Markdown

X Article 常把代码、JSON、数组、日志、字段树显示为普通段落。保存时必须改为 fenced code block，例如 `bash`、`json`、`text`。

章节标题如 `一、...`、`二、...`、`1. ...` 必须转成 `##` 二级标题，并在标题前后保留空行。

## 5. 回复容易和引用推文混淆

「热门回复（Top 10）」只记录页面真实可见的回复。不要把正文中的引用推文、嵌入推文、作者正文段落当成回复。

如果滚动后仍无法稳定展示回复，写：`当前页面未提取到可见回复。`

## 6. 视频默认不下载媒体

视频推文只发链接时，默认目标是拿字幕/逐字稿，不下载视频文件。只有用户明确说 `需要原视频`、`下载这个视频`、`把视频发我`、`同时下载视频` 时，才保存视频媒体。

## 7. Twitter 视频有可见字幕时先读网页

如果播放器显示 CC/字幕文本，先用主浏览器进度条逐点跳转采样可见字幕。不要直接跳到 ASR。

采样只记录 `{start, text}`；不保存视频文件；不导出 cookies/token/localStorage。

## 8. YouTube 原视频不能只搜一次

Twitter/X 视频常是 YouTube 原视频剪辑。没有 Twitter 字幕时，应根据网页标题、作者、外链、封面文字、专有名词多次查找 YouTube 原视频。

原视频可以比 Twitter 剪辑更长，但必须能解释剪辑关系。不能解释时，只能写成候选，不当作确定来源。

## 9. 命令行登录凭证必须单独确认

`yt-dlp` 需要登录或 unsupported 时，不自动使用 `--cookies-from-browser`。先用主浏览器可见内容完成文字保存；只有用户明确授权后，才考虑把浏览器登录态交给命令行工具。

## 10. 调试信息不要进笔记

保存到 Obsidian 的 Markdown 只能包含最终内容、媒体路径、摘要、回复和来源说明。不要写入报错堆栈、DOM 选择器调试、候选搜索过程或下载日志。
