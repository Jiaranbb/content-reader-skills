# 环境与运行

## 固定路径

路径写在 `config.json`，默认保存到：

- `/00-Inbox/Twitter/多媒体`
- `/00-Inbox/Twitter/创作素材`
- `/00-Inbox/Twitter/知识资源`

目录不存在时直接创建。

不要在执行过程中询问保存路径；需要调整时只改 `config.json`。

## 依赖

基础依赖：

- 浏览器能力：用用户主浏览器访问网页，读取页面可见内容。
- `yt-dlp`：公开媒体、字幕、元数据提取。
- `curl`：下载公开图片或媒体兜底。

可选依赖：

- `ffmpeg`：本地 ASR 前提取音频。
- `faster-whisper`：无字幕时的本地 ASR 兜底。

## 登录态规则

默认允许使用用户主浏览器网页中的登录可见内容：

- 可以读取页面上可见的正文、图片、视频和 X Article。
- 不输出、不保存、不复制 cookies、token、localStorage。
- 不自动使用 `yt-dlp --cookies-from-browser`。
- 不要求用户额外登录；如果当前主浏览器不可见，就说明限制。

如果页面内容不可见或 X Article 返回 403，停止并说明网页不可见限制。只有用户明确授权把浏览器登录态交给命令行工具时，才进入 `cookies-from-browser` 方案。

## 错误处理

| 场景 | 处理 |
|------|------|
| 网页不可见 / 跳登录墙 | 停止并说明网页不可见限制 |
| X Article 返回 403 | 停止并说明当前公开路径不可用 |
| yt-dlp Unsupported URL | 尝试网页可见内容；不可见则停止 |
| yt-dlp 需要登录 | 不自动读取 cookies；先保存网页可见内容，需要命令行登录态时单独确认 |
| 字幕不存在 | 如用户需要逐字稿，再考虑本地 ASR |
| ASR 依赖缺失 | 标注 `无字幕，且本地 ASR 依赖缺失` |
