# YouTube Reader JR 工作流

## 默认路径：只保存逐字稿

1. 识别输入链接类型。
   - 单视频：直接处理。
   - playlist / 合集：用 `yt-dlp --dump-single-json` 展开条目，一条视频生成一篇笔记。
2. 获取元数据：标题、作者、发布时间、时长、规范链接。
3. 获取字幕：
   - 参考 `podcast-transcript-txt` 的字幕链路，逐个窄语言请求，不把中英文混在同一轮。
   - 语言顺序：`zh-Hans -> zh-CN -> zh-Hant -> zh -> en-orig -> en`。
   - 遇到 `429 Too Many Requests` 时，先退避几秒并重试当前中文语言。
   - 只有中文都不可用时才退回英文。
   - 固定使用 `--extractor-args youtube:player_client=android`。
4. 清洗字幕：
   - 去掉 VTT 时间戳、HTML 标签、滚动字幕重复片段。
   - 保留字幕开始时间，逐字稿每行使用 `[HH:MM:SS]` 前缀。
   - 合并短句，保留可读段落。
5. 如果没有可用字幕，下载临时音频并用 `faster-whisper` 本地转写。
6. 生成「内容摘要」：
   - 必须根据完整逐字稿提炼 3-4 条核心观点/知识点。
   - 如果原字幕是英文，摘要也优先写成中文。
   - 不要按段落复述，也不要只摘录字幕原句。
   - 不允许留空，也不允许只写 `待整理`。
   - 自动摘要只作为兜底；如果内容泛泛，执行 agent 必须读逐字稿后人工改写。
7. 生成「划重点」：
   - 每条必须包含时间戳。
   - 时间戳后面必须写“从这里开始看什么”，例如某个概念解释、某个案例、某个论证段落、某个操作建议。
   - 不要只是复制该时间点字幕；要看该时间点后约 60-90 秒内容，提炼成可回看的观看导航。
8. 写入 Obsidian Markdown 笔记。

## 显式媒体下载路径

- `--action download-media`：只下载视频，不写笔记。
- `--include-media`：先写逐字稿笔记，再下载视频，并在笔记中加入 Obsidian 可预览的 `<video>` 标签。

## 批量处理规则

- 用户明确说「全部」时可以处理整个 playlist。
- 用户只说「这个合集」但没有说明范围，且条目较多时，先询问范围或建议 `--max-items`。
- 每条视频独立写一篇 Markdown，避免合集笔记过长。

## 登录与隐私边界

- 默认不使用浏览器 cookies。
- 不输出、不保存 cookies、token、localStorage。
- 如果 YouTube 返回登录、年龄、会员或地区限制，直接报告阻塞原因，不绕过限制。
