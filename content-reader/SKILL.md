# content-reader

这是一个组合 skill。它负责识别链接属于哪个平台，并把任务路由到对应的平台 skill。不要在这里实现平台细节。

## 总原则

1. 先识别平台，再识别用户意图。
2. 优先使用对应平台的专用 skill。
3. 不要自动使用任何平台的登录态、cookies 或主浏览器账号。
4. 如果目标平台需要登录、会员权限、私密权限或 cookies，停止并说明限制；只有用户明确授权时，才进入对应平台的登录态方案。
5. 多个链接混在一起时，按平台分组处理；同一平台内按用户意图批量执行。

## 平台路由

| 平台 | 链接特征 | 路由 |
|------|----------|------|
| 小红书 | `xhslink.com`、`xiaohongshu.com`、`explore/` | 使用 `xhs-reader` |
| Twitter/X | `x.com`、`twitter.com` | 使用 `twitter-reader` |
| YouTube | `youtube.com`、`youtu.be` | 使用 `youtube-reader` |
| B 站 | `bilibili.com`、`b23.tv` | 使用 `bilibili-reader` |
| 未知平台 | 其他 URL | 不猜测，先询问用户希望保存笔记、下载媒体还是提取正文 |

## 用户意图

| 意图 | 触发表达 | 处理 |
|------|----------|------|
| 保存笔记 | `保存一下`、`收藏`、`存一下`、只发链接 | 路由到平台 skill 的保存笔记流程 |
| 只下载视频 | `只下载视频`、`下载这个视频`、`把视频发我` | 路由到平台 skill 的只保存视频流程 |
| 只下载图片 | `下载图片`、`只要图片` | 路由到平台 skill 的只保存图片流程 |
| 保存笔记并下载原视频 | `需要原视频`、`原视频也需要`、`同时下载视频` | 路由到平台 skill 的保存笔记流程，并把原视频作为修饰需求 |
| 只拿逐字稿 | `拿逐字稿`、`提取字幕`、`转文字` | 路由到平台 skill 的字幕 / ASR 流程 |

默认规则：

- 只有链接、没有额外指令：默认保存笔记。
- 明确说「只下载」：不要生成 Markdown 笔记。
- 同时出现「保存」和「需要原视频」：保存笔记，并下载原视频。
- 同时出现多个平台：分别路由，不把不同平台内容合并成一篇笔记，除非用户明确要求合并。

## 小红书路由

使用 `xhs-reader`。

必须保留 `xhs-reader` 的账号安全规则：先打开网页检查登录态；检测到真实登录态就停止；只出现登录弹窗则关闭后继续；不使用小红书登录账号。

## Twitter/X 路由

使用 `twitter-reader`。

必须保留 `twitter-reader` 的安全规则：

- 默认先访问网页读取可见内容。
- 不输出、不保存、不复制 cookies、token、localStorage。
- 命令行 `cookies-from-browser` 只能在用户明确授权后使用。

## YouTube 路由

使用 `youtube-reader`。

必须保留 `youtube-reader` 的默认媒体规则：

- 默认只保存逐字稿笔记，不下载视频。
- 用户明确只要媒体时，只下载视频，不写 Markdown。
- playlist 条目很多且用户未给范围时，先确认范围或 `max_items`。
- 不默认读取浏览器 cookies；遇到私有、会员、年龄或地区限制时说明原因。

## B 站路由

使用 `bilibili-reader`。

必须保留 `bilibili-reader` 的默认媒体规则：

- 默认只保存逐字稿笔记，不下载视频。
- 用户明确只要媒体时，只下载视频，不写 Markdown。
- 分 P、合集或播放列表条目很多且用户未给范围时，先确认范围或 `max_items`。
- 不默认读取浏览器 cookies；遇到会员、登录或地区限制时说明原因。

## 输出

执行前简短说明路由结果：

```text
识别为：小红书
意图：保存笔记并下载原视频
路由：xhs-reader
```

执行后输出对应平台 skill 的结果，例如保存路径、下载路径、失败原因或需要用户补充的信息。
